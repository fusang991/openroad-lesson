# 4. 布局 (Placement)

## 原理

布局是将综合后的门级网表中的标准单元放置到芯片核心区域的物理位置。好的布局直接影响后续时钟树和布线的质量。

布局分为两个阶段：
1. **全局布局 (Global Placement)**：粗略确定每个单元的位置，允许重叠
2. **详细布局 (Detail Placement)**：消除重叠，将单元对齐到合法位置

## 输入文件

| 文件 | 来源 |
|------|------|
| `2_floorplan.odb` | 布局规划数据库 |
| `2_floorplan.sdc` | 时序约束 |

## 运行命令

```bash
util/docker_shell make do-place DESIGN_CONFIG=designs/nangate45/gcd/config.mk
```

## 子步骤详解

### 3_1_place_gp_skip_io — 全局布局（跳过 IO）

- 第一轮全局布局，不考虑 IO 位置
- 快速获得初始布局

### 3_2_place_iop — IO 布局

- 确定输入/输出端口的精确位置
- 根据时序要求优化 IO 排列

### 3_3_place_gp — 全局布局

- 使用 IO 位置信息重新全局布局
- 优化目标：最小化线长 + 满足时序
- 此步骤耗时最长（约 1 秒，消耗 284 MB）

### 3_4_place_resized — 单元调整

- 根据时序要求调整单元尺寸
- 对关键路径上的单元进行放大（upsize）
- 插入缓冲器改善信号完整性

### 3_5_place_dp — 详细布局

- 消除单元重叠
- 将单元对齐到标准单元行
- 优化局部线长

## 关键指标

布局完成后查看报告：

```
Design area 884 um^2 76% utilization.
```

| 指标 | 数值 |
|------|------|
| 设计面积 | 884 μm² |
| 利用率 | 76% |
| 标准单元数 | 781 (35 时序 + 382 组合 + 316 填充 + 48 Tap) |

## 输出文件

| 文件 | 说明 |
|------|------|
| `3_place.odb` | 布局完成的数据库 |

## 验证要点

- 无单元重叠（check_placement）
- 利用率合理
- 时序初步满足或接近满足
