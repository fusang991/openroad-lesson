# 概述

## 什么是 RTL 到 GDSII？

RTL 到 GDSII 是芯片物理设计的完整流程，将硬件描述语言（Verilog/VHDL）转换为可以送去制造的版图文件。

整个流程分为以下阶段：

```
RTL 代码 (Verilog)
    │
    ▼
┌─────────────┐
│  逻辑综合    │  Yosys: RTL → 门级网表
└─────┬───────┘
      ▼
┌─────────────┐
│  布局规划    │  OpenROAD: 确定芯片面积、IO 位置
└─────┬───────┘
      ▼
┌─────────────┐
│  电源网络    │  OpenROAD: 搭建供电网络 (PDN)
└─────┬───────┘
      ▼
┌─────────────┐
│  布局        │  OpenROAD: 将标准单元摆放到位
└─────┬───────┘
      ▼
┌─────────────┐
│  时钟树综合  │  OpenROAD: 插入时钟缓冲器，平衡时钟偏斜
└─────┬───────┘
      ▼
┌─────────────┐
│  布线        │  OpenROAD: 连接所有信号线
└─────┬───────┘
      ▼
┌─────────────┐
│  收尾        │  OpenROAD + KLayout: 填充、DRC、生成 GDSII
└─────────────┘
```

## 本教程使用的工具

| 工具 | 用途 | 版本 |
|------|------|------|
| **Yosys** | 逻辑综合 | ORFS 内置 |
| **OpenROAD** | 布局布线、时序分析 | ORFS 内置 |
| **KLayout** | GDSII 生成与查看 | ORFS 内置 |
| **Docker** | 运行环境 | 任意版本 |

## 本教程的设计案例

**GCD（最大公约数）模块**

- 源代码：757 行 Verilog
- 功能：计算两个 16 位整数的最大公约数
- 为什么选它：简单、完整、几分钟就能跑完全流程
- 工艺库：Nangate45（开源 45nm 工艺库）

## 如何使用本教程

1. 按顺序阅读每一章，每章对应流程的一个阶段
2. 每章包含：原理简介 → 实际命令 → 输入/输出文件 → 关键指标
3. 跟着环境搭建章节操作，你可以在自己的机器上复现全部结果
